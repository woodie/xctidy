# xctidy v0.2.0

## Features

- `xctidy --version` / `-v` -- prints the installed version and exits
  immediately, no stdin read required. The version comes from the
  nearest git tag (`git describe --tags --always --dirty`), so it always
  reflects what's actually installed rather than a hand-maintained
  constant. Output is a bare number with no `xctidy`/`v` prefix, matching
  `xcbeautify --version`'s style.
- `./test.sh` -- run a single spec file without hand-writing the
  `xcodebuild -only-testing:` invocation:

  ```
  ./test.sh EngineSpec
  ./test.sh Tests/XctidyKitTests/EngineSpec.swift   # tab-completed paths work too
  ```

  Accepts a bare class name, a `.swift` path, an already-qualified
  `Target/Class` filter, or any raw `xcodebuild` flag. Configuration
  (scheme, destination, test target) sits in environment-overridable
  variables at the top of the script for reuse in other SwiftPM packages.

## Under the hood

- CI now checks out full git history (`fetch-depth: 0`) so
  `git describe --tags` can actually see tags, plus a smoke-test step
  that verifies `--version` output on every run.
- Release badge link and linter config cleaned up; test-running
  instructions in the README/docs corrected.
