# xctidy v0.1.0

First tagged release of xctidy.

## Highlights

- Nested describe/context/it tree rendering for `xcodebuild test` output,
  cross-referencing your spec files to disambiguate comma-flattened test
  names (or falling back to a heuristic if no spec path is given).
- Three output styles: the default glyph format, `-fd` (RSpec doc-style),
  and `-fs` (Mocha spec-style) -- all ending in the same xcbeautify-style
  summary footer.
- Drop-in replacement for fastlane's `xcodebuild_formatter` (scan/gym/snapshot).
- Written in Swift, builds to a static binary via `make install`.

## Known limitations

- Only understands Quick/Nimble's `describe`/`context`/`it`, not Swift
  Testing's `@Test`/`@Suite` macros.
- Tree de-duplication isn't yet safe under
  `xcodebuild -parallel-testing-enabled`'s interleaved output.

See [docs/HOW_IT_WORKS.md](https://github.com/woodie/xctidy/blob/v0.1.0/docs/HOW_IT_WORKS.md)
for details on both.
